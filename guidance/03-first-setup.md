# 初次配置向导

> 首次安装后，JishuShell 会引导你完成初始配置。本文说明每个步骤的含义和操作方法。

---

## 设置管理员密码

首次访问 `http://localhost:8090` 时，会进入密码设置页面。

**要求：**
- 至少 8 个字符
- 两次输入必须一致

设置完成后，密码以 bcryptjs 哈希存储，不可逆。如需修改，参见 [系统维护 → 修改密码](10-system-maintenance.md#修改管理员密码)。

---

## 配置向导（7 步）

### 步骤 1：环境检测

检测以下组件是否已正确安装并运行：

| 组件 | 说明 |
|------|------|
| Node.js | 要求 ≥ 22.x |
| Docker | 容器运行时 |
| Nomad | Agent 调度引擎 |
| OpenClaw | AI Agent 运行时 |

每项显示版本号和状态（✅ 正常 / ⚠️ 需升级 / ❌ 未安装）。

### 步骤 2：Node.js 升级（如需）

若 Node.js 版本低于 22，可在此页面触发自动升级（通过 nvm 安装最新 LTS 版本）。

### 步骤 3：Docker 安装（如需）

若未检测到 Docker，点击**安装 Docker** 执行自动安装（需 sudo 权限，实时显示进度）。

### 步骤 4：Nomad 安装（如需）

若 Nomad 未就绪，自动下载并配置，完成后自动启动服务。

### 步骤 5：OpenClaw 安装（如需）

拉取 OpenClaw Docker 镜像或安装 npm 包。完成后显示当前版本号。

### 步骤 6：配置 AI 模型提供商

选择默认的 AI 模型上游提供商，所有实例可复用此配置。

**支持的提供商（部分）：**

| 提供商 | 说明 |
|--------|------|
| OpenAI | GPT-4o、GPT-4 Turbo 等 |
| Anthropic | Claude 3.5、Claude 3 系列 |
| Google | Gemini 1.5 Pro/Flash |
| DeepSeek | DeepSeek V3/R1 |
| Moonshot | Kimi 系列 |
| Ollama | 本地模型（无需 API Key）|
| 自定义 | 兼容 OpenAI API 的任意端点 |

**填写内容：**

- **API Key** — 从提供商官网获取
- **Base URL**（可选）— 自定义接口地址（用于代理或私有部署）
- **默认模型** — 选择或填写模型名称

> 💡 此配置不是必须的，可以跳过，在每个实例的配置页单独设置。

### 步骤 7：完成

显示各组件安装状态摘要，点击**进入面板**跳转到仪表盘。
