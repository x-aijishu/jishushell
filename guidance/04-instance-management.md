# 实例管理

> 介绍如何在 JishuShell 中创建、启停、克隆和删除 AI Agent 实例。

---

## 仪表盘概览

登录后首先看到的是**仪表盘**，顶部显示系统状态，下方列出所有实例。

### 系统状态卡片

| 卡片 | 含义 |
|------|------|
| 运行中 | 当前处于运行状态的实例数量 |
| CPU | 系统 CPU 使用率（%）|
| 内存 | 内存使用率（%）|
| 磁盘 | 磁盘使用率（%）|

> ⚠️ 资源使用率超过 90% 时卡片变为红色警告。

树莓派等设备还会显示 **CPU 温度**。

---

## 实例状态说明

| 状态 | 颜色 | 含义 |
|------|------|------|
| 运行中 | 绿色 | Agent 正常运行，可以对话 |
| 启动中 | 黄色（闪烁）| 正在启动，稍等片刻 |
| 已停止 | 灰色 | 已手动停止 |
| 失败 | 红色 | 启动或运行出错，查看日志排查 |
| 已崩溃 | 红色 | 进程异常退出 |

---

## 创建实例

1. 点击顶部 **+ 新建实例**
2. 填写表单：

| 字段 | 说明 |
|------|------|
| 实例名称 | 显示名称，可包含中文和特殊字符 |
| 实例 ID | 唯一标识符，仅含小写字母、数字和连字符，最长 63 位，**创建后不可修改** |
| 描述 | 可选，备注用途 |
| 克隆自 | 可选，从已有实例复制配置（不复制运行时数据）|

3. 点击**创建**

创建成功后：
- 若已配置默认模型提供商 → 自动跳转到 **Chat** 标签
- 若未配置 → 跳转到 **Config** 标签，提示先完成配置

### 实例 ID 规则

- 只允许：小写字母 `a-z`、数字 `0-9`、连字符 `-`
- 不能以连字符开头或结尾
- 最长 63 字符
- 示例：`my-assistant`、`work-bot-1`

---

## 启动 / 停止 / 重启

在仪表盘实例列表或实例详情页右上角，可执行：

| 操作 | 说明 |
|------|------|
| 启动 | 启动已停止的实例 |
| 停止 | 停止运行中的实例（保留配置和数据）|
| 重启 | 先停止再启动，用于配置生效 |

操作后约 3-10 秒内状态更新。

---

## 克隆实例

克隆会复制源实例的完整配置（`openclaw.json`），适合快速创建相似实例：

1. 点击 **+ 新建实例**
2. 在「克隆自」字段选择源实例
3. 填写新的名称和 ID
4. 点击**创建**

> ⚠️ 克隆只复制配置，不复制 IM 登录状态、Skills 安装状态、运行时数据。

---

## 删除实例

> ⚠️ 删除操作**不可逆**，会永久删除实例的配置和数据。

1. 进入实例详情页
2. 点击右上角 **···** 菜单 → **删除实例**
3. 在输入框中输入实例名称（精确匹配）确认
4. 点击**确认删除**

---

## 重启调度引擎

Nomad 是底层调度引擎，正常情况下无需手动操作。若遇到所有实例均无法启动的情况，可在仪表盘点击**重启引擎**。

> ⚠️ 重启引擎会**中断所有正在运行的实例**，请谨慎操作。

---

## 版本更新

面板检测到新版本时，会在仪表盘顶部显示橙色更新横幅：

```
发现新版本：0.1.0 → 0.2.0   [立即升级]
```

点击**立即升级**后，面板自动下载新版并重启，约 30 秒后手动刷新页面即可。

> 实例详情页也会显示相同的更新提示。
