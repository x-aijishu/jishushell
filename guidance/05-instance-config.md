# 实例配置参考

> 每个实例拥有独立的配置文件（`openclaw.json`），控制模型、渠道、代理等行为。

---

## 进入配置页

在实例详情页点击 **Config** 标签。

首次进入时，会显示当前实例的完整配置（JSON 格式）。

---

## 配置编辑方式

面板提供两种编辑模式，可随时切换：

### 表单模式（默认）

通过可视化表单填写各字段，适合日常配置修改。

### JSON 模式

直接编辑原始 JSON 文本，适合高级用户或批量修改。点击右上角 **JSON** 按钮切换。

## 主要配置项

### 模型代理配置

JishuShell 内置 LLM 代理，实例通过代理访问上游模型，无需在 OpenClaw 中直接配置 API Key。

```json
{
  "x-jishushell": {
    "proxy": {
      "upstream": {
        "provider": "openai",
        "apiKey": "sk-...",
        "baseUrl": "https://api.openai.com/v1"
      }
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `provider` | 提供商名称（openai / anthropic / google / deepseek 等）|
| `apiKey` | 上游 API Key，保存后加密存储 |
| `baseUrl` | 自定义接口地址（可选，用于中转代理）|

> 💡 API Key 在界面显示为 `(已保存, 留空保持不变)` 时，留空不填即保留现有值。

### 默认模型

```json
{
  "models": {
    "default": "claude-3-5-sonnet-20241022"
  }
}
```

设置实例启动时使用的默认模型。

## 保存配置

编辑完成后有两个保存选项：

| 按钮 | 说明 |
|------|------|
| **保存** | 仅保存配置，不重启（下次重启后生效）|
| **保存并重启** | 保存后立即重启实例，配置即时生效 |

> ⚠️ 离开配置页前若有未保存修改，会弹出确认提示。

## 实例日志

在 **Logs** 标签查看实例运行日志：

- 支持切换 `stderr` / `stdout` 日志类型
- 支持关键字搜索过滤
- 每 3 秒自动刷新，新日志到达时自动滚动到底部
- 最多显示最近 200 行

---

## 用量统计

在 **Usage** 标签查看：

- 请求总量（入站 / 出站 Token 数）
- 按模型提供商分类的用量
- 按渠道（飞书 / 微信 / 直连）分类的用量
