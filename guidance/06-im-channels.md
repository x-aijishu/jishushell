# IM 渠道接入

> JishuShell 支持将 AI Agent 接入飞书（Lark）和微信，直接在 IM App 中与 Agent 对话。

---

## 接入流程概览

```
打开快捷配置 → 选择 IM 标签 → 点击渠道
    ↓
检测插件是否已安装
    ├─ 未安装 → 自动运行安装命令 → 安装完成
    └─ 已安装 → 跳过安装
    ↓
生成二维码 → 扫码 → 绑定成功
```

## 飞书 / Lark 接入

### 前置条件

- 已有飞书企业账号（个人版也支持创建机器人）
- 网络可访问飞书 OAuth 服务

### 操作步骤

1. 在实例详情页，点击右上角 **⚡ 快捷配置**
2. 切换到 **💬 即时通讯** 标签
3. 找到「飞书 / Lark」，查看右侧状态徽标：
   - **安装插件** → 需先安装，点击后自动执行安装
   - **已安装** → 直接进入扫码

4. **安装插件**（首次）

   点击「安装插件」后，系统在实例环境中执行：
   ```
   npx -y @larksuite/openclaw-lark install
   ```
   安装过程约 10-30 秒，完成后自动继续。

5. **扫码绑定**

   安装完成后弹出二维码弹窗：
   - 使用**飞书 App** 扫描二维码
   - 在手机上点击确认授权
   - 面板检测到授权后显示「登录成功！」

6. 绑定成功后，向飞书机器人发消息即可触发 Agent。

### DM 配对审批

飞书机器人在接收私信（DM）时，会生成一个一次性**配对码**，需管理员审批后方可建立对话连接。

#### 在 Dashboard 中审批（推荐）

绑定飞书后，「DM 配对审批」区块会出现在快捷配置面板的飞书区域下方：

1. 点击 **View Pending** 按钮，拉取当前待审批请求
2. 请求列表中找到对应配对码（如 `LVU7PNYK`）
3. 点击旁边的 **Approve** 按钮，审批成功后面板显示绿色提示

#### 通过命令行审批

```bash
# 查看待审批请求（自动检测运行中的实例）
jishushell pairing list

# 审批指定配对码（默认 channel=feishu）
jishushell pairing approve LVU7PNYK

# 完整形式（多实例时需指定 --instance）
jishushell pairing approve feishu LVU7PNYK --instance my-instance
```

> **多实例说明：** 只有一个运行中的实例时，`--instance` 可省略；存在多个运行实例时必须指定。

### 飞书机器人配置（企业自建）

若需要使用企业自建应用（而非个人扫码），需要在飞书开放平台创建应用并配置：

- 回调地址：`http://<你的服务器地址>/feishu/callback`
- 开启机器人能力和消息订阅

具体参考飞书开放平台文档。

## 微信接入

### 前置条件

- 微信账号（个人号即可）
- 网络可访问微信 OAuth 服务

### 操作步骤

1. 在实例详情页，点击 **⚡ 快捷配置** → **💬 即时通讯**
2. 找到「微信」，查看右侧状态徽标：
   - **安装插件** → 首次使用需安装
   - **已安装** → 直接扫码

3. **安装插件**（首次）

   系统自动执行：
   ```
   npx -y @tencent-weixin/openclaw-weixin-cli install
   ```
   安装过程约 10-60 秒（依网速而定）。

4. **扫码绑定**

   弹出二维码后：
   - 使用**微信**扫描
   - 在手机上确认登录
   - 面板显示「登录成功！」

5. 邀请机器人后，向它发消息即可与 Agent 对话。

### 多账号说明

- 每个 JishuShell 实例可绑定**一个**微信账号
- 若需多账号，创建多个实例分别绑定

---

## 二维码过期处理

飞书和微信二维码均有时效（约 3-5 分钟）。过期后：

1. 弹窗提示「二维码已过期」
2. 关闭弹窗，重新点击渠道按钮
3. 获取新二维码

## 常见问题

### 安装插件失败

**可能原因：**
- 实例未启动（需先启动实例）
- npm 网络访问受限

**解决方法：**
1. 确认实例状态为「运行中」
2. 在实例 **Logs** 标签查看详细错误
3. 配置 npm 镜像后重试：
   ```
   /config npm set registry https://registry.npmmirror.com
   ```

### 扫码后无响应

- 确认手机和服务器网络均正常
- 飞书需确认已在手机端点击「确认授权」
- 尝试关闭弹窗重新扫码

### 绑定成功后收不到消息

检查实例是否正在运行（Chat 标签显示绿色运行状态）。若服务停止，重新启动实例即可恢复消息接收。
