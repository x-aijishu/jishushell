# MCP 配置

> 通过 MCPorter 管理 MCP 服务器，让 Agent 能够调用外部工具和服务，大幅扩展能力边界。

---

## 核心概念

| 术语 | 说明 |
|------|------|
| **MCP** | Model Context Protocol，工具调用标准协议 |
| **MCPorter** | MCP 服务器管理工具，负责连接和维护 MCP 服务 |
| **MCP 服务器** | 提供具体工具的独立服务（如查询火车票、搜索等）|

---

## 第一步：安装 MCPorter

MCPorter 是管理 MCP 服务器的必要工具，需先安装。

1. 打开实例详情页 → **⚡ 快捷配置** → **🔌 常用 MCP**
2. 顶部显示 MCPorter 状态：
   - **需要安装** → 点击「安装 MCPorter」
   - **已安装** → 跳过此步骤

3. 点击**安装 MCPorter**，命令注入到对话框：
   ```
   安装 https://clawhub.ai/steipete/mcporter
   ```
4. 等待 Agent 完成安装

安装完成后，刷新快捷配置面板，MCPorter 状态变为「已安装」。

## 添加 MCP 服务器

### 方式一：使用预设（推荐）

面板内置了常用 MCP 服务器预设，点击即可一键配置：

| 预设 | 说明 | 配置方式 |
|------|------|---------|
| **12306-MCP 车票查询** | 查询中国铁路 12306 车票信息 | `npx -y 12306-mcp` |

**操作步骤：**

1. 在 **🔌 常用 MCP** 标签中找到目标预设
2. 查看右侧徽标：
   - **一键配置**（蓝色）→ 点击自动写入配置
   - **已配置**（绿色）→ 已完成，无需重复操作
3. 点击后配置自动写入 `workspace/config/mcporter.json`

### 方式二：粘贴自定义 MCP JSON

适合添加任意符合 MCP 规范的服务器：

1. 在 **🔌 常用 MCP** 标签中展开「自定义配置」输入框
2. 粘贴 JSON 格式的 MCP 服务器配置：

```json
{
  "my-tool": {
    "command": "npx",
    "args": ["-y", "my-mcp-package"]
  }
}
```

或 HTTP 模式：

```json
{
  "my-tool": {
    "baseUrl": "http://localhost:3100"
  }
}
```

3. 点击**添加到配置**

## 查看已配置的服务器

「MCPorter 已配置」区域列出所有已添加的 MCP 服务器（预设中已显示的服务器不在此重复显示）。

每行显示：

| 信息 | 说明 |
|------|------|
| 服务器名称 | 配置中的 key |
| 传输方式 | 命令行参数或 HTTP 地址 |
| 状态徽标 | 检测中… / 已配置 / 正常 · N工具 / 错误 |

**状态说明：**

| 状态 | 含义 |
|------|------|
| 检测中… | 正在连接服务器，获取工具列表 |
| 已配置 | 配置已保存，但暂无 live 状态 |
| 正常 · N工具 | 服务正常，有 N 个可用工具 |
| offline / error | 服务不可用，检查后端服务是否运行 |

## 删除 MCP 服务器

1. 在「MCPorter 已配置」列表中找到目标服务器
2. 点击右侧**删除**按钮
3. 确认后，该服务器从 `mcporter.json` 中移除

## MCP 配置文件

MCP 配置保存在实例目录中：

```
~/.jishushell/instances/<id>/openclaw-home/.openclaw/workspace/config/mcporter.json
```

文件结构：

```json
{
  "mcpServers": {
    "12306-mcp": {
      "command": "npx",
      "args": ["-y", "12306-mcp"]
    }
  }
}
```

## 常见问题

### 服务器状态一直显示「检测中…」

MCP 服务器需要时间启动（尤其是 `npx` 方式，首次运行需下载包）。等待 10-30 秒后刷新快捷配置面板。

### 服务器状态显示 `offline`

1. 确认 `npx` 命令可以正常运行（检查 Node.js 和网络）
2. 在实例 **Logs** 标签搜索 MCP 相关错误
3. 尝试删除并重新添加该服务器

### 添加的配置未生效

确认配置写入后重启实例：实例详情页 → 点击**重启**按钮。
