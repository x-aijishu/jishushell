# 系统维护

> 介绍日常维护操作，包括版本更新、密码管理、系统监控、日志查看和用量统计。

---

## 版本更新

### 自动检测

JishuShell 会在**仪表盘**和**实例详情页**自动检测新版本。有新版本时，页面顶部显示橙色横幅：

```
发现新版本：0.1.0 → 0.2.0   [立即升级]
```

### 一键升级

点击**立即升级**后：

1. 后台自动执行 `npm install -g jishushell@latest`
2. 面板服务自动重启（约 30 秒）
3. 手动刷新浏览器页面，升级完成

> ⚠️ 升级过程中面板暂时不可访问，**正在运行的实例不受影响**（由 Nomad 独立管理）。

### 手动更新

```bash
npm install -g jishushell@latest
systemctl restart jishushell
```

## 修改管理员密码

1. 点击左侧导航**设置**
2. 在「修改密码」区域填写：
   - 当前密码
   - 新密码（至少 8 位）
   - 确认新密码
3. 点击**保存**

> 修改成功后会话令牌失效，下次访问需重新登录。

## 系统监控

在**设置页**的「系统状态」区域可查看实时监控数据：

| 指标 | 说明 | 警告阈值 |
|------|------|---------|
| CPU 使用率 | 当前 CPU 占用百分比 | > 90% 变红 |
| CPU 温度 | 仅树莓派等支持温度检测的设备显示 | — |
| 内存使用 | 已用 / 总量（MB）及百分比 | > 90% 变红 |
| 磁盘使用 | 已用 / 总量（GB）及百分比 | > 90% 变红 |

仪表盘也会以卡片形式汇总显示这些指标，每 10 秒自动刷新。

### 资源不足时的处理建议

**内存不足**
- 停止暂时不用的实例
- 减少同时运行的实例数量

**磁盘不足**
- 清理旧的 Docker 镜像：`docker image prune -a`
- 删除不再使用的实例

**CPU 过高**
- 检查是否有实例在执行高负载任务
- 重启对应实例

## 查看实例日志

在实例详情页点击 **Logs** 标签：

### 日志类型

| 类型 | 说明 |
|------|------|
| `stderr` | 标准错误输出（默认），包含错误和调试信息 |
| `stdout` | 标准输出，包含 Agent 运行日志 |

### 操作

- **搜索过滤**：在搜索框输入关键字，实时过滤匹配行
- **复制日志**：点击右上角「复制」按钮，复制过滤后的可见日志
- **自动刷新**：每 3 秒自动拉取最新日志；若接近底部，自动滚动

> 每次最多显示最近 200 行。如需更多历史日志，参考以下命令：

```bash
# 查看实例完整日志文件
cat ~/.jishushell/instances/<instance-id>/openclaw-home/.openclaw/logs/agent.log
```

## 查看用量统计（开发中）

在实例详情页点击 **Usage** 标签：

### 统计内容

| 维度 | 说明 |
|------|------|
| 总入站 Token | LLM 代理收到的请求 Token 总量 |
| 总出站 Token | LLM 代理转发给上游的 Token 总量 |
| 按提供商 | 各上游服务商的用量分布 |
| 按渠道 | 飞书 / 微信 / 直连接口的用量分布 |

### 用量监控建议

- 定期查看用量，防止 API Key 超出配额
- 若某渠道（如飞书）用量异常高，检查是否有误发送的消息流

## 重启调度引擎

若出现所有实例无法启动或调度异常，可尝试重启 Nomad 引擎：

1. 在**仪表盘**点击「重启引擎」按钮
2. 确认操作

或通过命令行：

```bash
systemctl restart nomad
```

> ⚠️ 重启引擎会**中断所有正在运行的实例**，实例会在引擎就绪后自动重新调度。

## 备份实例配置

重要配置建议定期备份：

```bash
# 备份单个实例配置
cp ~/.jishushell/instances/<id>/openclaw-home/.openclaw/openclaw.json \
   ~/backup-openclaw-<id>-$(date +%Y%m%d).json

# 备份全部实例
tar -czf ~/jishushell-backup-$(date +%Y%m%d).tar.gz \
    ~/.jishushell/instances/
```

> MCP 配置（`mcporter.json`）和 Skills 目录也在实例 `workspace/` 下，上述备份命令一并包含。
